# 04. 会員登録フロー

## フロー全体
```
LP/CTA → [メール入力] → [プロフィール入力] → [規約同意] → [決済（年3万円）] → 登録完了メール → サポーター活動開始
```

## 現行の登録プロセス（移行前）
1. 入会エントリーフォーム（Google Forms）の提出
2. 説明会（個別またはグループ）に参加
3. 入会手続き完了後、各種活動へ参加可能
- 問い合わせフォーム: https://forms.gle/P6hWCc1NShYQNysi8

### 課題
- Google Formsベースのため決済連携がない
- 手動での対応が必要（理事の工数がかかる）
- 説明会ステップが導入障壁になっている可能性

## 新ツールで目指すフロー（設計案）
```
[LP] → [サポーター申込フォーム]
         ├─ メールアドレス入力
         ├─ 氏名 / ふりがな
         ├─ 所属（法人の場合は会社名 ※任意）
         ├─ 電話番号
         ├─ 規約同意チェック
         └─ [決済画面へ進む]
                ↓
         [決済完了]
                ↓
         [自動: 登録完了メール送信]
         [自動: コミュニティ招待リンク送付]
         [自動: 管理DBに会員データ登録]
```

## 取得項目
### 必須（公式サイトの問い合わせフォーム準拠）
- 氏名
- 氏名（ふりがな）
- メールアドレス
- 電話番号

### 任意
- 所属（法人の場合は会社名）
- Web掲載名（サポーター一覧に載せる名前 ※掲載は任意）

### 運営側で管理
- 会員ID
- 登録日
- 会員区分（サポーター）
- ステータス（有効 / 停止 / 退会）
- 決済ID（プロバイダ側の顧客ID）
- 次回課金日

## 本人確認（KYC）
- **不要**（一般社団法人の会員制度であり、金融サービスではないため）

## メール送信
| タイミング | 内容 |
|---|---|
| 登録完了時 | ウェルカムメール（特典一覧 / コミュニティ参加方法 / 次回公開収録案内） |
| 更新1ヶ月前 | 更新リマインド |
| 更新完了時 | 更新完了通知 |
| 決済失敗時 | カード更新依頼 |
| 退会時 | 退会確認＋お礼 |

## 公開収録の参加ルール（登録フロー関連）
| 対象 | 登録要否 | 条件 |
|---|---|---|
| 挑戦サポーター | 会員登録+決済済 | 毎回無料 + 3名招待可 |
| 若者（18〜29歳） | **登録不要** | 事前申込（Peatix等）のみ |
| サポーターの招待者 | 登録不要 | サポーターからの招待 |
| 上記以外 | — | **参加不可** |
